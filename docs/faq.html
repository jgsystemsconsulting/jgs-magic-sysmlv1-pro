§01 · Common questions

I installed the pro extension but the pro tools do not appear. What is wrong?

The pro JAR is loaded by the free base plugin at CATIA Magic startup, so it only takes effect after a full restart. Close CATIA Magic completely and reopen it. If the tools are still missing, confirm the free base plugin is installed and that your licence file is present and valid.

The pro extension seems to be installed, but it silently does nothing. Why?

The pro JAR cannot run on its own. It is loaded dynamically by the free base plugin (the SysML v1 MCP Bridge), so without that base plugin present the JAR is never loaded and nothing happens. Install the free base plugin first, then restart CATIA Magic.

Where does the licence file go, and what do I do if it is not detected?

Place the licence file in the location given with your licence (the bridge reads it locally at startup). If it is not detected, check that the file is the unmodified one we issued, that its name and path match the instructions, and that CATIA Magic has been restarted since you added it. Then confirm Pro is active by checking the licence status reported by the bridge.

I get a SHA-256 digest mismatch on the JAR. What does that mean?

The base plugin verifies the pro JAR against a known SHA-256 digest before loading it. A mismatch means the JAR on disk is not the exact file we published: it is incomplete, modified, or corrupted in transit. Re-download jgs-sysmlv1-pro.jar, verify its SHA-256 against the published value in RELEASE-INFO.txt, and replace the copy on disk.

Does anything leave my machine? Can I run this air-gapped?

No data leaves your machine. The bridge, the pro JAR, and the licence check all run locally inside CATIA Magic, with no cloud service and no licence server to contact. It works fully air-gapped.

Which CATIA Magic versions and prerequisites are required?

You need a supported install of CATIA Magic (Cameo Systems Modeler) with the SysML plugin, the free SysML v1 MCP Bridge base plugin installed, and the matching pro extension JAR for your bridge version. The base plugin and the pro JAR must be from the same release so their digest check lines up. For exact supported versions, see the release notes shipped with your bridge.
